Ziona HMS: Automatic Camp & Database Sync

Instructions for setting up zero-click automated background data syncing on the customer's server PC.

1 Architecture & Data Flow Overview

The synchronization pipeline runs bidirectionally between your local database and the cloud:

[Public Camp Portal] -> Patients fill form on cloud site -> Saved directly to Neon Cloud DB (as 'pending_sync')
[Local Server Pull] -> Downloads 'pending_sync' -> Assigns Local Patient ID -> Saves to local DB
[Local Server Push] -> Extracts local DB state -> Uploads mirror to Neon Cloud DB (marks as 'synced')

2 Local Server Environment Configuration

Ensure the connection strings are properly configured in your local environment file:

  1. Open C:\2035-HMS\SAAS_ERP\.env in a text editor.
  2. Verify or add the correct connection strings for both local and cloud databases:
# The local database connection string (hospital intranet)
DATABASE_URL="postgresql://postgres:your_local_password@localhost:5432/revive_hms?schema=public"

# The hosted Neon Cloud database connection string (Singapore pooler link)
CLOUD_DATABASE_URL="postgresql://postgres:your_cloud_password@sg-pooler.neon.tech/revive_db?sslmode=require"

# The branch/tenant identifier
NEXT_PUBLIC_TENANT_ID="00000000-0000-0000-0000-000000000001"

3 Verify the Batch Script

The orchestration script coordinates the pull-then-push workflow. Open C:\2035-HMS\SAAS_ERP\SYNC_TO_CLOUD.bat and verify it contains:

@echo off
cd /d "C:\2035-HMS\SAAS_ERP"

:: 1. Pull cloud registrations
node scripts/pull_camp_registrations.js
if %errorlevel% neq 0 (
    echo [ERROR] Pull sync failed. Aborting push to protect local database.
    exit /b %errorlevel%
)

:: 2. Push local mirror to cloud
node scripts/smart_sync.js

4 Set Up Windows Task Scheduler (Zero-Click Automation)

Configure Task Scheduler to run the sync script in the background every 5 minutes:

A. Open Task Scheduler
Press Win + R, type taskschd.msc, and hit Enter.
B. Create Task Settings (General Tab)
Click Create Task... in the Actions pane. Set the following:
  • Name: Ziona HMS Data Sync
  • Select Run whether user is logged on or not.
  • Check Run with highest privileges.
C. Schedule Settings (Triggers Tab)
Click **New...**. Set:
  • Begin the task: On a schedule (or At startup)
  • Check Repeat task every: Select 5 minutes.
  • Set for a duration of: Select Indefinitely.
D. Execution Settings (Actions Tab)
Click **New...**. Set:
  • Action: Start a program
  • Program/script: Browse and select C:\2035-HMS\SAAS_ERP\SYNC_TO_CLOUD.bat
  • Start in (optional): Enter C:\2035-HMS\SAAS_ERP (CRITICAL - DO NOT LEAVE BLANK)

5 Verification and Testing

Verify that the zero-click sync is working correctly:

  1. Register a test patient named "Auto Sync Test" on the portal: https://revive-qqom.vercel.app/camp-registration
  2. Wait 5 minutes for the scheduler to trigger.
  3. Open your local PostgreSQL client (like pgAdmin) and query: 
    SELECT id, patient_number, first_name, status, created_at 
FROM "hms_patient" 
ORDER BY created_at DESC LIMIT 3;
  4. The patient "Auto Sync Test" should automatically appear locally with a sequence ID (e.g. `PAT-xxxxx`).
